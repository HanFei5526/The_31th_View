> 12页 | 5分钟 | 面向科技/投资背景产品评委（观众展示与实机演示在单独环节）

**全局视觉风格**：与游戏整体调性协调（古雅、沉静、留白），与所用图片的水墨/园林风格统一。具体配色、字号、动画等由实现时自由发挥，大纲只规定内容层次和布局结构。文字为主，图片为辅，不使用全屏背景图。

**主题**：第一印象，建立调性

- 标题下方：副标题「AI驱动的文人园林叙事解谜游戏」，中等字号，字间距略宽

- 整体入场动画：模糊渐现（blur-in），各元素依次延迟出现

## 第2页 · 破题（30秒）

**主题**：为什么做这个——痛点与立论

**布局**：
- **上部**：核心立论，大字居中
  - 「不是寻找一幅失踪的画，而是找回一种被遮蔽的观看方式。」
- **过渡**：「传统文化数字化体验存在三重断层」，小字灰色
- **下部三列卡片**：每张卡片粗体关键词 + 一行对比句
  - 卡片1：**叙事空心化** — 只有知识，没有情感。多数文化产品堆砌年表与术语，用户看完记不住任何一个人的故事。
  - 卡片2：**AI标签化** — 只有聊天，没有机制。AI角色沦为百科问答，对话脱离游戏进程，无法形成有意义的互动循环。
  - 卡片3：**视角单一化** — 只有名家，没有被遮蔽的人。反复消费同一批名人IP，真正参与创作却未留名的人从未进入叙事。

**演讲节奏**：先念大字停顿一拍，再快速过三个断层，落点在"被遮蔽的人"——引出下一页。

## 第6页 · 核心玩法（25秒）

**主题**：游戏怎么玩

**标题**：「核心玩法循环」

**布局**：四步流程横排卡片，纯文字+序号，不放截图。逐个浮入动画（stagger-list）。

- 每步结构：序号（01-04，朱红色） + 粗体动词 + 说明句（灰色小字）
  - **01 探索** — 在画中世界的园林场景中寻找呼吸光斑，点击触发隐藏线索与空间记忆。
  - **02 收集** — 获取断簪、残砚、草图拓片等关键信物，逐步构成指向王蘅身份的证据链。
  - **03 研讨** — 回到工作室，在AI修复笔记本中串联线索，用自己的语言表达对真相的理解。
  - **04 复原** — 终章以四道核心问题引导玩家做出最终判断，还原五百年前被遮蔽的观看方式。

## 第7页 · AI设计理念（30秒）

**主题**：AI在游戏中扮演什么角色，为什么这样设计

**标题**：「AI不是噱头」

**布局**：左文右图（约55:45比例），文字为主、图片为辅

- **左侧**：
  - 大字定性：「AI不是自由聊天窗口」
  - 副句：「而是被放进"修复笔记本"这一叙事媒介中的推理引擎。」
  - 两个对比块：
    - 「常见做法」（如AI Dungeon / Character.AI）：AI = 开放对话，玩家可以问任何问题 → 容易跑题、剧透、破坏叙事节奏

  - 底部强调句（强调色）：「AI服务叙事，而不是替代叙事。」

## 第8页 · AI设计决策（35秒）

**主题**：我们为什么这样设计AI——每个选择背后的权衡

**标题**：「AI设计决策」

**布局**：三个设计决策纵向排列，每条讲清"选了什么 + 为什么"

- 每条结构：色块标识 + 粗体决策点 + 设计理由
  1. **进度约束的知识注入** — 按章节/进度筛选知识片段注入AI prompt，未解锁内容AI无法触及。**为什么**：叙事解谜游戏的核心体验是"逐步发现"，防剧透是硬需求，不能交给AI自行判断边界
  2. **规则判定而非AI判定** — 研讨环节用关键词规则判定玩家推理，不依赖AI做对错裁判。**为什么**：游戏进程需要确定性，LLM判定存在随机性风险，规则判定保证每个玩家体验一致
  3. **离线预写降级** — AI服务不可用时自动切换预写回复池。**为什么**：完整游戏体验不能依赖第三方API的可用性，AI是增强层而非基础层
- **底部展望**（小字灰色）：「下一阶段：扩展为五种文学声音（教授 · 批注 · 笔记本 · 旁注 · 报告），各有独立知识边界与叙事功能」
- **可选补充**：简单流程示意图：`玩家输入 → 进度过滤 → 知识注入 → AI生成 → 回复约束`

## 附：修订记录（基于三视角质疑审核）

> 经专业评委、资深玩家、普通观众三视角质疑后，结合"5分钟面向产品评委"的定位做出以下修订：

| 修改项 | 修改内容 | 修改原因 |
|--------|----------|----------|
| 第4页 故事内核 | 20s → 15s；删去"她不求被正名"副句；新增一句王蘅身份铺垫 | 面向产品评委不宜过度渲染情感；但需让评委理解"玩家在追寻什么" |
| 第7页 AI对比块 | 「常见做法」中点名竞品（AI Dungeon / Character.AI） | 产品评委需要竞品坐标系，一句话定位 |
| 第8页 标题与定位 | "AI机制创新" → "AI设计决策"；三个要点改为"选了什么+为什么这样选"的权衡式表述 | 评委会看穿简单方案的过度包装；诚实展示设计权衡反而更有说服力 |
| 第9页 底部文案 | "不追求大众爆款" → 引用同类作品（山海旅人/笼中窥梦/画中世界）作为市场锚点 | 空洞的定位语缺乏说服力，需要可参照的市场依据 |
| 第10页 商业模式 | 第三阶段从实线节点改为虚线/灰色"远期探索" | 学生团队谈景区合作/IP授权缺乏可信度，降级处理避免减分 |
| 第12页 结语 | 删去第二句结语，只保留一句；第二句改为口头补充 | 收尾干脆，避免文字堆砌削弱力度 |
| 全局 | 受众描述更正为"面向产品评委"；移除对普通观众的术语降级考虑 | 观众展示与实机演示在单独环节 |
| 全局 | 移除所有具体色值、字号、CSS属性 | 只规定内容层次和布局结构，视觉风格交由实现时自由发挥 |
