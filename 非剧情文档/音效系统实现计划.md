# 音效系统实现计划

## Context

项目当前完全无音频系统。设计文档中仅提到三个关键音效（墨迹流动声、灰泥剥落声、扫描仪声），标注"可简化"。本计划为游戏搭建完整的音频架构，覆盖 BGM、环境音、UI 交互音效三层，并实现双世界（现实/画中）音效风格差异化。

**用户选择**：BGM + 环境音 + UI音效 / Howler.js + 占位音频 / 双世界差异化风格

## 架构概览

```
SoundManager (singleton) ──┬── BGM 层 (crossfade切换)
                           ├── Ambient 层 (循环环境音)
                           └── SFX 层 (一次性触发)
```

集成方式：SoundManager 监听 GameEngine 事件总线自动切换 BGM/环境音，组件内直接调用 `playSFX()` 触发交互音效。

## 新建文件

### 1. `Web/src/core/sound-manager.js` — 核心音频管理器

单例模块，API：

- `init(engine)` — 绑定引擎事件，设置 autoplay unlock
- `playBGM(key, { fade })` — 按 key 查配置播放，自动 crossfade 当前 BGM
- `stopBGM({ fade })` — 淡出停止
- `playAmbient(key, { fade })` / `stopAmbient({ fade })` — 环境音管理
- `playSFX(name)` — 按名称触发一次性音效
- `toggleMute()` / `setMuted(bool)` — 全局静音
- `setMasterVolume(v)` / `setBGMVolume(v)` / `setAmbientVolume(v)` / `setSFXVolume(v)`

内部机制：
- Howl 实例缓存（path → Howl），避免重复创建
- crossfade：旧轨淡出 + 新轨淡入同时进行（1500ms）
- autoplay unlock：document 首次 click/touchstart/keydown 时 `Howler.ctx.resume()`
- localStorage 持久化音量偏好（key: `sanyijing-audio-prefs`）

### 2. `Web/src/data/sound-config.js` — 音频配置数据

三个映射表：

- `bgm` — 场景/世界组合 → 文件路径（如 `'prologue-real' → '/audio/bgm/prologue-workshop.mp3'`）
- `ambient` — 环境音名 → 文件路径
- `sfx` — 音效名 → 文件路径
- `sceneMap` — 场景名 → `{ bgm, ambient }` 自动选择映射

双世界差异化：
- 现实世界：安静工作室嗡鸣、机械微声、晚间虫鸣
- 画中世界：鸟鸣风声、流水、古琴单音、笔墨质感

### 3. `Web/public/audio/` — 音频资源目录

```
audio/
  bgm/          (8 files: prologue + ch1~3 × 2worlds + finale)
  ambient/       (5 files: workshop-hum, workshop-evening, garden-birds, water-stream, rain-pavilion)
  sfx/           (24 files: ui-click, typewriter-tick, clue-found, ink-spread, etc.)
```

占位策略：每个文件放一个极短的静音/微弱音调 MP3（SFX=0.1s, BGM/ambient=2s），确保播放逻辑可验证，后续逐个替换真实音频无需改代码。

## 修改文件

### 4. `Web/package.json` — 添加 howler 依赖

```
"dependencies": { "howler": "^2.2.4" }
```

### 5. `Web/src/main.js` — 初始化音频

在 `gameEngine.init(app)` 之后加：
```js
import soundManager from './core/sound-manager.js';
soundManager.init(gameEngine);
```

### 6. `Web/src/components/hud-bar.js` — 添加静音按钮

在 HUD 按钮栏增加音效开关按钮（♪ / 已静音图标），点击调用 `soundManager.toggleMute()`。

### 7. `Web/src/components/narration-bar.js` — 打字机音效

在 `_typeChar` 中每隔 3 个字符调用一次 `soundManager.playSFX('typewriter-tick')`，避免过于密集。

### 8. `Web/src/components/notebook-floating.js` — 面板交互音效

- `expand()` → `playSFX('notebook-open')`
- `collapse()` → `playSFX('notebook-close')`
- tab 切换 → `playSFX('tab-switch')`
- 发送消息 → `playSFX('chat-submit')`

### 9. `Web/src/components/inventory-popup.js` — 物件匣音效

- `open()` → `playSFX('ui-open')`
- `close()` → `playSFX('ui-close')`

### 10. `Web/src/components/painting-viewer.js` — 古画交互音效

- 线索命中（金色涟漪）→ `playSFX('clue-found')`
- 点击未中（灰色涟漪）→ `playSFX('clue-miss')`

### 11. `Web/src/core/scene-manager.js` — 转场音效

- `switchWithInkSpread()` → `playSFX('ink-spread')`
- `switchWithScroll()` → `playSFX('scroll-unfurl')`

## 事件驱动集成（在 SoundManager.init 内部）

| 引擎事件 | 响应 |
|---------|------|
| `scene-enter` | 根据 sceneMap 自动切换 BGM + ambient |
| `world-changed` | 切换对应世界的 ambient |
| `item-collected` | `playSFX('item-collect')` |
| `gate-completed` | `playSFX('gate-complete')` |

| Window CustomEvent | 响应 |
|-------------------|------|
| `clue-collected` | `playSFX('clue-found')` |
| `tool-used` | `playSFX('tool-activate')` |
| `all-tools-used` | `playSFX('convergence')` |

---

## Crossfade 时序

- BGM 切换：1500ms crossfade（匹配世界转换 overlay 时长）
- Ambient 切换：1200ms fade-in / 800ms fade-out
- UI SFX：即时触发，无 fade
- 转场 SFX：在转场 overlay 动画开始时触发

## 实施顺序

1. `npm install howler`
2. 创建 `sound-config.js`（纯数据）
3. 创建 `sound-manager.js`（核心逻辑）
4. 创建占位音频文件到 `public/audio/`
5. 修改 `main.js` 接入
6. 修改 `hud-bar.js` 加静音按钮
7. 逐个组件添加 `playSFX` 调用（narration-bar → notebook → inventory → painting-viewer → scene-manager）
8. 端到端测试

## 验证方式

1. 启动 `npm run dev`，打开浏览器
2. 点击任意位置确认 autoplay unlock（控制台无 AudioContext 警告）
3. 进入序章 → 确认 BGM + 环境音自动播放
4. 打字机对话 → 听到间隔 tick 声
5. 发现线索 → 金色音效 / 点空 → 灰色音效
6. 打开笔记本/物件匣 → 开关音效
7. 跌入画中 → BGM/ambient 自动切换为画中世界风格
8. 点击静音按钮 → 全部音频立即静音，刷新后偏好保持
9. 返回菜单再进入 → 音量设置从 localStorage 恢复
