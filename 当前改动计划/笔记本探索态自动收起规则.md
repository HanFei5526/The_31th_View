# 笔记本探索态自动收起规则

## 背景

当前修复笔记本会在系统提示提到“笔记本 / 记录页 / 对话页 / 线索”等内容时自动弹开，这个行为保留。问题在于：很多系统提示结束后，场景已经进入可探索状态，画面中出现可点击光点，但笔记本仍保持展开，容易挡住玩家继续点击场景。

## 目标

保留“记录提示时自动弹开笔记本”的反馈感，同时在玩家需要回到场景探索时自动收起笔记本，把操作焦点还给画面。

## 规则

1. 线索记录、物件获得、记录更新等提示出现时，笔记本可以自动弹开。
2. 当提示结束后进入“点击场景光点 / 点击画面元素继续探索”的状态时，笔记本自动收起。
3. 以下阶段不自动收起：
   - 轻量讨论阶段
   - 综合研讨阶段
   - 终章记录回顾阶段
   - 序章工具检测阶段，因工具区依附在笔记本内，需要保持展开
4. 玩家手动打开笔记本查看时，不额外强制关闭；自动收起只放在明确回到探索态的场景流转点。

## 拟调整范围

- `Web/src/pages/chapter1-paint.js`
  - 兰雪堂入场提示结束并显示热点前收起。
  - 兰雪堂探索完成，显示前往缀云峰光点前收起。
  - 缀云峰提示后、芙蓉榭倒影谜题开始前收起。
  - 线索记录后若回到继续探索状态，收起；断簪轻量讨论保持展开。

- `Web/src/pages/prologue.js`
  - 综合研讨通过后，汇聚点击点出现前收起。
  - 序章工具检测与线索扫描阶段仍保持展开。

- `Web/src/pages/chapter2-paint.js`
  - 远香堂入场提示结束并显示热点前收起。
  - 诗词比对开始前收起。
  - 小飞虹入场提示结束并显示热点前收起。
  - 水面回声、残砚记录结束后回到可探索状态时收起。
  - 旧批注后的轻量讨论保持展开。

- `Web/src/pages/chapter3-paint.js`
  - 南厅、北厅、留听阁入场提示结束并显示热点前收起。
  - 渗字、封墙线条、红线显现、草图拓片等提示结束后，如接下来需要点击场景元素，收起。
  - 王蘅的信后进入轻量讨论，保持展开。

## 验收方式

1. 运行基础构建检查。
2. 不做复杂浏览器测试。
3. 代码检查确认：
   - 记录提示仍可弹开笔记本。
   - 进入探索态前有明确收起。
   - 轻量讨论、综合研讨、终章回顾不被误收起。
