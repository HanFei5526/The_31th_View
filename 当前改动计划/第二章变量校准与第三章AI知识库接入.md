# 第二章变量校准与第三章AI知识库接入

## 背景

当前修复笔记本后端已完成序章、第一章，以及第二章部分知识片段接入。合并主线章节实现后，第二章、第三章和终章页面已存在真实玩法流程，其中第二章/第三章已经调用 `queryNotebook()`，但后端知识库仍未完整覆盖第三章。

## 改动目标

1. 校准第二章知识片段的解锁条件，使其严格对应现有页面中的真实进度变量。
2. 新增第三章知识片段，覆盖鸳鸯馆南厅、北厅、留听阁、草图拓片、王蘅的信、匾额追认等阶段。
3. 扩展章节边界 prompt，使序章至第三章的剧透限制按当前章节动态变化。
4. 终章暂不生成报告，保持当前“回顾记录、不开放终章自由问答”的方向。

## 实施范围

- `Web/server/index.js`
  - 更新 `NOTEBOOK_KNOWLEDGE` 中第二章片段的解锁条件。
  - 追加第三章片段。
  - 调整 `buildNotebookSystemPrompt()` 的章节边界文本。

## 不做内容

- 不改终章报告生成。
- 不改变统一回复格式：仍为 `周老师批注：...`。
- 不改变“未找到相关记录”的兜底规则。
- 不改前端 UI。

## 验证方式

1. `node --check Web/server/index.js`
2. `npm run build`
3. 通过接口构造不同 progress，检查知识片段是否按变量解锁。
