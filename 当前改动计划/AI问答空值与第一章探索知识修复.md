# AI问答空值与第一章探索知识修复

## 背景

试玩中发现：

1. 序章普通问答可能出现只有 `周老师批注：` 的空回复。
2. 第一章回复有时过长，被 UI 截断。
3. 第一章探索阶段，玩家针对当前场景现象提问时，后端因相关知识尚未解锁而回复“未找到相关记录”。

## 目标

1. 后端对空回复或只有前缀的回复进行兜底，统一返回 `周老师批注：翻了翻，没有找到相关记录。`
2. 适度提高后端返回 token，同时在 prompt 中要求短句和完整收束，减少半截回复。
3. 第一章前端向后端传入当前子场景状态，后端据此解锁场景观察类知识。
4. 第一章兰雪堂和芙蓉榭探索阶段可回答“我该留意什么”“倒影为什么多出东西”“水面光线能否点击”等辅助推断问题，但仍不提前透露断簪和后续真相。

## 改动范围

- `Web/src/core/ai-service.js`
  - 支持页面传入额外 AI 上下文。

- `Web/src/pages/chapter1-paint.js`
  - 调用 `queryNotebook()` 时传入当前第一章子场景状态与谜题状态。

- `Web/server/index.js`
  - 后端上下文接收 `sceneState`。
  - 新增第一章场景观察类知识片段。
  - 增加空回复规范化。
  - 调整 `max_tokens` 与 prompt 输出约束。

## 非目标

- 不改变 UI 布局。
- 不改谜题操作流程。
- 不提前解锁断簪、“蘅”字或后续章节真相。

## 验证

1. `npm run build` 通过。
2. `node --check Web/server/index.js` 通过。
3. 搜索确认兜底文本统一。
