# 序章开场节奏：画面显现与笔记本获得时机修正

## 问题

序章进入工作室后，字幕在背景画面尚未完全显现时就开始播放；同时修复笔记本入口和快捷问题在开场一开始就可见，和台词逻辑不一致。

## 修改范围

- `Web/src/pages/prologue.js`

## 修改方式

- 序章进入后先等待工作室背景图加载，并等待首屏渲染稳定，再开始播放第一句旁白。
- 初始挂载修复笔记本和 HUD，但保持隐藏。
- 第一句旁白结束后，再授予“修复笔记本”，显示 HUD/笔记本入口，并开放对应快捷问题。
- 已完成序章后的自由探索分支仍直接显示笔记本和 HUD。

## 验收标准

- 序章未完成时，开场第一句旁白播放前不显示笔记本入口。
- 第一句旁白结束后才出现“获得物件：修复笔记本”和笔记本入口。
- `npm run build` 通过。
