# 死代码清理：废弃组件与样式精简

## 背景

对当前 `Web/src` 做引用关系检查后，发现 UI 重构前遗留的若干组件和样式已经不再被入口或场景导入。分章节实现文档也明确标注这些旧组件已由 `narration-bar`、`notebook-floating`、`hud-bar`、`inventory-popup`、`painting-viewer` 替代。

## 本轮删除范围

### 删除确认废弃组件

- `Web/src/components/prologue-dock.js`
- `Web/src/components/scanner-ui.js`
- `Web/src/components/chat-panel.js`
- `Web/src/components/clue-explorer.js`
- `Web/src/components/clue-popup.js`
- `Web/src/components/notebook-panel.js`

### 删除确认废弃样式/入口

- `Web/src/styles/prologue-dock.css`
- `Web/src/style.css`

### 清理旧样式块

- `Web/src/styles/index.css`
  - 删除旧 `ScannerUI` 相关 `.scanner-*` 样式块。
  - 删除旧 `ChatPanel` 相关 `.chat-*` 样式块。
  - 删除旧 `NotebookPanel` 相关 `.notebook-ai-panel` / `.notebook-toggle` / `.notebook-overlay` / `.notebook-query-*` 等样式块。
- `Web/src/styles/painting-viewer.css`
  - 删除旧 `CluePopup` 弹窗样式块。

## 本轮保留范围

- `Web/src/components/gate-panel.js`
- `Web/src/styles/gate-panel.css`

原因：`GameEngine` 当前仍实例化 `GatePanel`，`main.js` 仍导入 `gate-panel.css`。虽然界面已迁移到 `NotebookFloating`，但该路径仍是运行时对象，不在本轮删除。

## 验收标准

- `npm run build` 通过。
- 搜索确认废弃组件文件名不再存在于 `Web/src/components`。
- 搜索确认旧组件专属样式选择器无残留，或仅存在于文档/计划说明中。
- 不删除当前场景正在导入的组件与样式。

- 已删除 2 个确认无入口引用的旧样式/入口文件：`styles/prologue-dock.css`、`src/style.css`。
