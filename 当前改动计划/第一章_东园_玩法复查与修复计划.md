# 第一章·东园：玩法复查与修复计划

通过审查试玩视频和代码比对，我发现当前的实现版本与《场景制作》和《玩法设计》文档存在多处出入。以下是详细的差异清单与修复方案：

## 1. 交互热点（Hotspot）坐标偏差严重

在视频中，玩家为了点击正确的物品（特别是兰雪堂的匾额和缀云峰的石缝），必须点击屏幕上非常偏离实际画面的位置。代码中的坐标完全没有对齐文档设定。

**修复方案：严格按照《场景制作》文档更新坐标**
*   **场景1（兰雪堂）**
    *   青石板路：`this._createHotspot(50, 80)` -> 修正为 `(50%, 85%)`
    *   翠竹（柳树）：`(35, 40)` -> 修正为 `(15%, 50%)`
    *   廊柱：`(75, 35)` -> 修正为 `(70%, 55%)`
    *   匾额（核心叙事点）：代码中居然是 `(75, 30)`，完全偏右！-> 修正为画面中央的 `(50%, 22%)`
    *   前进指示箭头：代码 `(80, 70)` -> 修正为中央 `(50%, 70%)`
*   **场景2（缀云峰）**
    *   石缝（低视角伏笔）：代码中是 `(70%, 75%)`，偏右下。-> 修正为左侧低处 `(30%, 72%)`
    *   前进指示箭头：代码 `(80, 65)` -> 修正为 `(75%, 65%)`
*   **场景3（芙蓉榭）**
    *   真实栏杆：确认是否为 `(50%, 35%)`
    *   断簪（翻转前）：确认是否为 `(45%, 72%)`
    *   断簪（翻转后）：确认是否为 `(45%, 28%)`

## 2. 芙蓉榭翻转谜题（缺少专属的翻转背景图）

*   **当前现象**：视频中 2:34 触发翻转时，只是通过 CSS 的 `transform: rotateX(180deg)` 简单粗暴地将同一张图片（`芙蓉榭_3.png`）倒转过来。导致倒影变成了正像，但依然带着水面的朦胧感，而真正的建筑被倒了过来。
*   **文档要求**：文档中明确要求：“特别要求：需提供翻转版本 `chapter1-furong-flipped.png`：以水面线为轴翻转后的画面，倒影变为正像，断簪变为实体清晰状态”。
*   **修复方案**：在 CSS 翻转动画执行到一半（或翻转完成时），必须通过 JS 将背景图从 `chapter1-furongxie.png` 瞬间替换为 `chapter1-furong-flipped.png`。
*   > [!IMPORTANT]
    > **需要确认**：目前资源库 `TestPic/` 中似乎还没有 `chapter1-furong-flipped.png`。请问这张“清晰倒影”的翻转版图是否已经制作完成并上传？如果还没有，我需要等你上传后才能彻底修复这个翻转逻辑。目前我只能先写好切换图片的占位代码。

## 3. 场景5（返回工作室）的背景图错误

*   **当前现象**：视频 3:25 返回现实工作室时，周鹤年说话的背景图居然是一幅巨大的古画（第三十一景）。
*   **文档要求**：文档规定章末返回现实时，应“复用序章工作室背景 (`prologue-bg.png`)”。
*   **修复方案**：修改 `chapter1-workshop.js` 中的背景图地址，将 `this._bgImage = '/images/chapter1-bg-placeholder.png'` 更改为正确的 `'/images/prologue-bg.png'`。
