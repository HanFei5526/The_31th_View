# AI回复格式回收：周老师批注与通用修复知识

## 背景

当前后端 prompt 允许“资料页批注、夹页便签、研讨摘记”等多样化形式。但最新设定要求修复笔记本始终表现为“周老师预先留下的批注”，不可像实时 AI 一样临场判断；不能回答的内容仍应表现为“翻找不到记录”。

## 目标

1. 所有 AI 回复统一使用 `周老师批注：` 开头。
2. 删除“批注来源多样化”的输出要求。
3. Prompt 增加“通用修复知识”可答范围，允许回答工具用途、装裱、题签、边注、题跋等专业基础问题。
4. 剧情线索仍严格按当前已解锁知识回答；未发现线索、后续章节、王蘅身份、最终真相一律回复未找到相关记录。

## 改动范围

- `Web/server/index.js`
  - 更新 `buildNotebookSystemPrompt()`
  - 增加通用修复知识块
  - 统一兜底文本为 `周老师批注：翻了翻，没有找到相关记录。`

- `Web/src/pages/chapter1-paint.js`
  - 统一第一章轻量讨论硬编码开场为 `周老师批注：`

- `Web/server/README.md`
  - 更新接口回复格式与约束说明

## 非目标

- 不修改 UI 结构。
- 不改研讨门槛通过判定逻辑。
- 不把语义判断改成后端硬分类器，本次仍通过 prompt 约束模型。

## 验证

1. `npm run build` 通过。
2. `node --check Web/server/index.js` 通过。
3. 搜索确认后端 prompt 不再要求多样化来源模板。
