## 硬性约束（审批后实施必须遵守）

1. **低视角复原资产统一按 16:9 制作**
   - `scan-painting.png`、`finale-view31-low.png`、`finale-view31-line.png`、`finale-painting-complete.png` 的核心画面统一按 16:9 视觉比例控制。
   - 其中 `finale-view31-line.png` 与 `finale-view31-low.png` 必须同尺寸、同构图、同裁切、同视角，不允许一张带边框、一张不带边框导致上色错位。
   - 建议以当前常用宽幅规格为基准，例如 `2750 x 1536` 或等比例 16:9 规格；最终以同一批输出资产的实际尺寸为准。

1.1. **线条密度不得继续增加**
   - `finale-view31-line.png` 的细密程度是后续资产的线条密度上限。
   - 设色层、完成态、扫描件可弱化、柔化或局部淡化线条，但不得新增更细碎、更密集的线描纹理。
   - 若图像生成导致建筑、树丛、竹叶或水纹线条明显变密，应退回重做或局部修正。

2. **复原中间态只使用无边框裸图**
   - `finale-view31-line.png` 作为黑白线稿底图，只包含第三十一景低视角画面本体。
   - `finale-view31-low.png` 作为彩色叠加层，也只包含同构画面本体。
   - 云纹锦边、册页装裱、案台质感只出现在 `finale-painting-complete.png`，避免局部填色时把装裱边框一起“复原”。

3. **终章画布容器必须匹配 16:9**
   - `.finale-canvas-container` 不再使用正方形 `50vmin x 50vmin` 作为最终实现。
   - 代码实现时应改为 16:9 宽幅容器，例如使用 `aspect-ratio: 16 / 9`，并配合 `width`、`max-width`、`max-height` 保证桌面与移动端都不裁切、不拉伸。
   - 线稿底图和着色层统一使用相同的 `background-size` / `background-position`，确保像素级对齐。

4. **局部着色遮罩以不穿帮为优先**
   - 初版可以沿用四块 `clip-path` 分区显色逻辑。
   - 如果美术资产完成后发现小飞虹、远香堂倒影、梧竹幽居竹影不适合简单四象限分区，应改为更贴合景物位置的局部遮罩（例如更精确的 `clip-path: polygon(...)` 或多层局部遮罩）。
   - 完成标准不是“分了四块”，而是玩家肉眼看起来像景物被逐步复原，不能出现明显矩形割裂或景物错位。

5. **转场状态类必须补齐**
   - 需要为大地图层、位置名牌层、低视角画布容器分别补齐淡出 / 淡入状态类。
   - `fade-out`、`visible`、`revealed`、`full` 等状态必须在 CSS 中有明确过渡，不只在 JS 中添加 class。
   - Q2 答对后应先让宏观定位信息退场，再淡入低视角线稿，避免地图与第31景画布长时间重叠造成视觉混乱。

6. **完成态必须与复原画面无跳变衔接**
   - `finale-painting-complete.png` 的画心内容必须与 `finale-view31-low.png` 同源，只是在外层增加册页锦边和装裱环境。
   - 从 Q4 完整着色到场景3完成图时，画心主体位置、比例和视角应保持一致，允许增加装裱边框，但不允许换构图。
