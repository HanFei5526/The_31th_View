# 批注模板与第一章AI接口：后端扩展

## 背景

当前后端已提供序章修复笔记本接口，但回复仍可能出现“周老师批注：”“笔记本页面上出现……”等舞台说明。用户希望回复不固定模板，但必须体现“正在翻阅周老师留下的批注、摘录或便签”，同时允许代码范围扩展到第一章。

## 目标

1. 调整后端 prompt，使 AI 回复呈现为修复笔记本中的既有记录、页边批注、夹页便签、研讨摘记或参考摘录。
2. 禁止“周老师批注：”、括号动作描写、实时对话式表述。
3. 后端知识库扩展到第一章，按第一章进度解锁匾额、低处观察、断簪、“蘅”字、章末现实讨论等内容。
4. 保持前端 `AIService` 与 `NotebookFloating` 调用方式不变。

## 改动范围

- `Web/server/index.js`
  - 将序章知识常量扩展为通用笔记本知识常量
  - 新增第一章知识片段与解锁条件
  - 更新 system prompt 的输出格式规则
  - 保留 `deepseek-v4-flash` 与 `/api/notebook/message`

- `Web/server/README.md`
  - 补充回复格式约束和第一章支持说明

## 非目标

- 不修改 UI。
- 不新增第一章强制研讨门槛。
- 不改变第一章剧情与谜题流程。
- 不提前透露第二章、第三章、终章真相。

## 验证

1. `npm run build` 通过。
2. `node --check Web/server/index.js` 通过。
3. 后端 prompt 中明确禁止舞台动作和“周老师批注：”前缀。
