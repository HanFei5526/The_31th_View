# 第三章封墙物件选择交互修正方案

## 问题现象

第三章留听阁封墙谜题中，第二次点击封墙后会出现“选择要使用的物件：”提示，并显示“残砚”“断簪”两个按钮。此时玩家点击场景内其他光点、背景区域或返回以外的场景元素都没有反应，容易误以为交互失效。

## 原因定位

当前逻辑在 `chapter3-paint.js` 的 `_showItemSelectOverlay()` 中创建了 `.ch3-item-select-overlay`，该浮层覆盖整个留听阁子场景。

对应样式中 `.ch3-item-select-overlay.active` 设置了 `pointer-events: auto`，因此透明区域也会接管鼠标事件。玩家看到的是透明背景，但实际点击会先落到这个全屏浮层上，底下的封墙、窗格等热点无法收到事件。

此外，当前取消选择依赖对话框 `playLine()` 结束后返回 `cancel`，但界面没有明确的“取消 / 暂不使用”按钮，也没有点击空白处退出的反馈，所以体验上像是被锁住。

## 修正目标

1. 保留封墙谜题的物件选择流程：第一步用残砚显现红线，第二步用断簪剥离灰泥。
2. 让玩家明确知道当前处于“选择物件”状态。
3. 提供可见、可靠的退出方式，避免误判为场景坏掉。
4. 不改动剧情基准、线索链或章节节奏。

## 实际改法

1. 去掉全屏透明浮层，只保留非阻塞的物件按钮组。
2. 物件按钮组本身可点击，但按钮组外的场景区域不再拦截鼠标事件。
3. 点击叙事框或小三角时取消物件选择。
4. 点击其他热点时先收起物件选择，再执行该热点自己的功能。
5. 选择残砚或断簪后，仍按原谜题顺序进入对应反馈或揭示流程。

## 涉及文件

- `Web/src/pages/chapter3-paint.js`
- `Web/src/styles/chapter3.css`

## 验收方式

1. 进入第三章留听阁，第一次点击封墙记录隐藏线条。
2. 第二次点击封墙出现物件选择按钮组。
3. 物件选择出现时，窗格与封墙热点仍可点击。
4. 点击叙事框或小三角后，物件选择关闭。
5. 选择残砚后红线显现；再选择断簪后进入草图揭示流程。
