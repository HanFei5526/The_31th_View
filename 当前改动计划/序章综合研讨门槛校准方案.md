# 序章综合研讨门槛校准方案

## 一、调整目标

序章的综合研讨必须是强制判定门槛。玩家需要主动表达出对三处线索关系的理解，才能出现“确认这个推断”并推进剧情。

本次调整目标是：

1. 保留“必须讨论并通过判定才能继续”的结构。
2. 避免快捷问题直接给出通关答案。
3. 收紧判定条件，减少空泛表达、否定表达、误解表达导致的误通过。
4. 限制 AI 批注越界，避免提前说出完整结论或透露后续信息。
5. 只修序章综合研讨，不影响第一章及后续章节的轻量讨论。

## 二、最高门槛与知识边界

这一节是本次修改的上位标准。后续快捷问题、判定关键词、AI prompt 都必须服从这里的边界。

### 1. 玩家通过序章综合研讨时，最多需要推理出的信息

序章综合研讨的最高通关门槛应停在"人为遮蔽"：

> 三处痕迹不是孤立破损，也不是画心丢失或重画的证据；它们共同说明：这页画的来源说明、边注记录或观看来源痕迹，曾在后来的装裱、配边或整理过程中被系统性地压住、裁去或排除。

其中最重要的是玩家表达出"不是自然破损，而是人为遮蔽 / 后来处理造成的遮蔽"。如果玩家已经围绕三处痕迹说清这一点，就可以通过；"来源说明"可以用自然说法表达，不要求固定术语。

换成玩家可自然表达的过关理解，就是：

1. 旧题签残角、"……所见"残字、底层细线三者存在关联。
2. 这些痕迹都位于画心之外、边缘、装裱层或底层，不像普通画面破损。
3. 它们更像原本用于说明这页画的来源、来历、旁注或观看记录的材料。
4. 这些说明性痕迹被后来的处理统一遮蔽或排除，说明问题出在"来源说明被处理掉"，不是"画本身不存在"。

玩家不必使用固定词，只要表达出以上意思即可。可以说"来源""出处""来历""旁注说明""这页画从哪里来的记录""谁所见的说明"等。

### 2. 通过门槛不要求玩家推理出的信息

序章不要求、也不允许玩家在这里被引导到以下结论：

1. 不要求说出"王蘅"。
2. 不要求判断王蘅是观看者。
3. 不要求知道低位视角来自芙蓉榭栏杆下。
4. 不要求知道远香堂、小飞虹、梧竹幽居三景同入一眼。
5. 不要求知道文徵明采用了王蘅的视角。
6. 不要求知道这些来源痕迹全部是王蘅本人留下。
7. 不要求判断后人遮蔽的具体动机，只需知道这是后来的整理、装裱或归档处理造成的遮蔽。

### 3. AI 在序章综合研讨中绝对不能泄露的信息

AI 批注在玩家通过前，绝对不能说出或暗示以下内容：

1. 不能提到"王蘅"，也不能说"某位女性""她""被遮蔽的观看者"等足以指向后续身份的说法。
2. 不能说第三十一景的低位视角来自某个人。
3. 不能说"文徵明保存了她的眼睛"或任何等价表达。
4. 不能说文徵明采用了别人的草图、构图、观看位置。
5. 不能说王蘅留下了题记、辅助线、记号或草图。
6. 不能说"芙蓉榭下""栏杆下""低处观看""三景同入一眼"等后续章节和终章答案。
7. 不能说"第三十一景不是文徵明画的""多人执笔""失踪画作""重画替换"等错误方向。
8. 不能编造当前记录外的馆藏记录、来源编号、数据库条目、历史档案或鉴定结论。
9. 不能在玩家通过前直接给出完整通关句，例如"有人系统性地遮蔽了这幅画的来源信息"。

### 4. AI 可以说的信息

AI 批注可以在序章综合研讨中说：

1. 三处痕迹都在边缘、装裱层、底层或画心之外。
2. 它们更像说明性材料的残留，而不是画面主体破损。
3. 旧题签、残字、底层细线可以并列比较。
4. 可以追问玩家："这些痕迹原本可能在说明什么？"
5. 可以提醒玩家："不要急着找具体姓名，先判断被处理掉的是哪一类信息。"
6. 可以指出："现有证据只能支持来源说明被处理过，还不能支持具体人物判断。"

### 5. 过关后的系统确认语边界

通过后出现的确认语可以收束为：

> 对。现在能确认的不是画心丢失，也不是画被重画，而是这页画的来源说明被后来的整理和装裱处理遮蔽了。三处痕迹共同指向这一点。

通过后的结论仍不应提到王蘅、芙蓉榭、低位视角来源、文徵明采用视角等后续答案。

## 三、当前问题

### 1. 快捷问题泄露答案

当前序章综合研讨快捷问题为：

- 三处痕迹都被处理过，是巧合吗？
- 被遮蔽的会不会是来源信息？
- 有人刻意抹去了什么证据吗？

问题：

- 第二条直接说出“来源信息”。
- 第三条直接说出“有人刻意”“抹去”“证据”。
- 玩家只点击前两条快捷问题，就可能触发通过。

### 2. 判定关键词过宽

当前关键词中有一些词单独命中时过于宽泛：

- “有人”
- “信息”
- “记录”
- “证据”
- “痕迹”
- “三处”
- “三个”
- “都是”

这些词容易让空泛表达误通过，例如：

- “记录里有信息” + “这三个东西都是线索”
- “三处痕迹都很奇怪” + “有人留下了痕迹”

### 3. 否定表达会被错误累积

当前综合判定会跨轮累积概念，但没有排除否定句。

风险示例：

- 第一轮：“不是有人故意遮蔽来源”
- 第二轮：“我觉得只是自然磨损”

第一轮虽然是否定判断，但仍命中“有人 / 故意 / 遮蔽 / 来源”等概念，后续可能误通过。

### 4. AI 批注会提前给结论

测试中 AI 对模糊输入会直接提到“来源记述”“这幅画从哪里来的信息接口”等内容，已经替玩家说出了关键判断。

对“是不是王蘅画的？”这类越界问题，AI 还可能提到“王蘅”或编出当前证据外的馆藏记录，这违反序章知识边界。

## 四、拟修改范围

### 1. `Web/src/core/gate-config.js`

调整 `gate_prologue_synthesis`：

- 修改 `quickThoughts`，改为观察整理型问题。
- 修改 `fallbackQuickThoughts`，多轮失败后给更强提示，但不直接形成“一键通关答案”。
- 收紧 `concepts` 关键词，删除或限制过宽词。
- 必要时新增更细的概念分组，让通过条件更像“理解表达”，而不是“关键词碰撞”。

建议方向：

快捷问题改为类似：

- “旧题签、残字和细线是不是都在画面边缘或底层？”
- “这三处痕迹有没有可能原本都在说明同一件事？”
- “如果不是自然磨损，它们更像是哪类处理留下的？”

这些问题帮助玩家整理证据，但不直接说出“来源信息被系统遮蔽”。

### 2. `Web/src/core/discussion-gate.js`

调整综合研讨判定过程：

- 如果玩家当前输入包含明显否定表达，本轮不累积通过概念。
- 如果玩家已经进入否定方向，AI 只做纠偏，不让这一轮触发通过。
- 保留 `minRounds: 2`，防止一轮直接通过。

建议处理：

- 对综合研讨单独加入否定保护。
- 否定保护只影响综合研讨，不改变其他线索辅助讨论。

### 3. `Web/src/core/ai-prompts.js`

收紧 `gate_prologue_synthesis` 的 AI 批注规则：

- 明确禁止在未通过前直接说出完整结论：
  - “有人系统性遮蔽来源信息”
  - “来源信息被抹去”
  - “这幅画从哪里来被处理掉”
- 明确禁止提到“王蘅”。
- 明确禁止编造馆藏记录、来源编号、外部档案等当前记录没有的信息。
- AI 只能围绕已见三处线索做观察引导。

同时修正一条通用规则：

- 当前 prompt 中写着“只讨论当前这一条线索，不要主动把多条线索串联到一个结论里”，这对综合研讨不合适。综合研讨本来就需要比较三处线索，应改成“可以比较三处线索，但不能替玩家说出最终结论”。

## 五、建议判定标准

玩家要通过序章综合研讨，至少应表达出两层意思：

1. 三处线索之间不是孤立偶然，而是存在共同处理或共同指向。
2. 被处理掉的不是画心本身，而是与这幅画的来源、说明、来历、观看记录有关的内容。

可以通过的表达示例：

- “这三处不像自然损坏，更像被人统一处理过，目的是让后人看不到这幅画来源相关的说明。”
- “旧题签、残字和底层线都不是画面本身，它们像是原本解释这页来历的东西，被重新装裱时压住或删掉了。”
- “它们共同指向同一件事：有人不是改画，而是在处理关于这幅画出处和观看来源的痕迹。”

不应通过的表达示例：

- “这三个东西都是线索。”
- “记录里有信息。”
- “有人留下了痕迹。”
- “是不是王蘅画的？”
- “这幅画是不是假的？”
- “不是有人故意遮蔽来源，我觉得只是自然磨损。”

## 六、测试计划

只做基础测试，不做复杂浏览器测试。

### 1. 判定脚本测试

用本地脚本直接测试综合门槛判定：

- 正确表达应通过。
- 当前快捷问题组合不应直接通过。
- 空泛表达不应通过。
- 否定表达不应通过。
- 越界猜测不应通过。

### 2. AI 回复抽样测试

通过本地 AI 后端抽测以下输入：

- “这三个东西都是线索。”
- “它们好像不是偶然。”
- “是不是王蘅画的？”
- “是不是这幅画是假的？”

检查 AI 是否：

- 不提王蘅。
- 不编外部资料。
- 不直接说出最终通关结论。
- 能把玩家拉回旧题签、残字、底层细线三处证据。

### 3. 构建检查

在 `Web/` 下运行基础构建检查：

```bash
npm run build
```

## 七、验收标准

完成后应满足：

1. 序章仍然必须通过综合研讨才能推进。
2. 快捷问题只能帮助观察和整理，不会直接喂出完整答案。
3. 玩家只点快捷问题不会自动通关。
4. 空泛表达不会误通过。
5. 否定句不会被当成有效理解累积。
6. AI 不会在未通过前替玩家说出最终结论。
7. AI 不会提到王蘅或后续章节信息。
8. 第一章和后续轻量讨论不受影响。

## 八、不纳入本次修改

以下内容本次不处理：

- 第一章断簪轻量讨论逻辑。
- 第二章、第三章轻量讨论。
- 笔记本自动展开 / 收起规则。
- 序章单条线索发现后的辅助讨论流程。
- 浏览器内完整通关回归测试。
