# AI 知识边界与空回复加固方案

## 背景

本次检测确认 AI 主链路可用，但存在三处需要加固的问题：

1. 终章结案笔记偶尔会生成边界不稳的表述，例如把“低位视角来源痕迹指向王蘅”说成“观看位置不属于文徵明”，或补写未出现的笔迹、画稿风格证据。
2. 笔记本查阅偶尔会暴露内部知识片段编号，例如 `ch2_huafeiyiren`。
3. 后端上游偶尔返回空内容时，当前接口仍可能按成功响应返回空字符串。

## 修改范围

- `Web/src/core/ai-prompts.js`
- `Web/src/core/knowledge-base.js`
- `Web/src/core/ai-service.js`
- `Web/server/index.js`

## 实施内容

1. 笔记本查阅 Prompt 增加约束：不得引用内部资料编号，只能自然转述已解锁资料。
2. 知识片段注入格式改为通用资料页编号，避免把代码内部 `id` 暴露给模型。
3. 终章结案笔记 Prompt 增加限制：不得新增笔迹、落笔习惯、文徵明其他画稿等未记录证据；不得使用会否定画心作者归属的简化表述。
4. 终章结案笔记无效输出过滤增加禁句，覆盖“观看位置不属于他”“文徵明其他画稿”“仅由一人之眼”等风险表达。
5. 后端代理在 DeepSeek 返回空内容时返回 502，交给前端重试或兜底。

## 验证方式

1. 运行前端构建检查。
2. 启动后端代理，检查 `/api/health`。
3. 抽样验证：
   - 序章诱导追问王蘅，不应提前泄露。
   - 第二章询问“画非一人”，不应暴露内部资料编号。
   - 终章结案笔记不应出现禁句或新增证据。
