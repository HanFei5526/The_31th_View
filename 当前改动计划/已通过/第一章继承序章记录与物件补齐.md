# 第一章继承序章记录与物件补齐

## 修改目标

- 第一章进入时先继承序章已获得的线索记录，再继续追加第一章新线索。
- 物件匣同步继承序章获得的「修复笔记本」。
- 修复笔记本物件只出现在物件匣，不作为记录页顶部批注显示。
- 保持笔记本「对话」仍按章节隔离，不把序章对话带入第一章。

## 涉及文件

- `Web/src/core/game-engine.js`
- `Web/src/pages/chapter1-paint.js`
- `Web/src/pages/chapter1-workshop.js`

## 主要改法

- 在游戏引擎中增加后续章节的前序成果补齐逻辑。
- 根据进入章节自动补齐序章关键进度、记录页条目和修复笔记本物件。
- 清理旧存档中已经写入记录页的「获得了物件 [修复笔记本]」批注。
- 在第一章场景挂载笔记本与物件匣前执行补齐，确保 UI 首次打开时就是完整状态。
- 第一章回到工作室后的场景也复用同一补齐入口，避免跨场景状态断开。

## 完成标准

- 第一章记录页能看到序章线索和本章新增线索。
- 第一章物件匣能看到序章获得的「修复笔记本」。
- 记录页顶部不再显示「沈念批注：获得了物件 [修复笔记本]」。
- 章节间对话记录仍然独立。
- 前端构建通过。
