# 章节入场等待与笔记本记录继承

## 修改目标

1. 第一章以及后续章节进入时，应先等入场转场遮罩消失、场景背景图加载完成，再显示对话框并开始文字打字。
2. 修复笔记本“记录”页应继承之前章节发现的线索；“对话”页按章节保存，换章后不继承上一章对话。

## 涉及文件

- `Web/src/pages/chapter1-paint.js`
- `Web/src/components/notebook-floating.js`

## 主要改法

- 第一章开场 `_startSequence()` 前增加 `_waitForSceneReady()`，等待兰雪堂背景图、浏览器渲染帧、入场 overlay 移除。
- `NotebookFloating` 挂载时从 `engine.notebookRecords` 渲染全局记录。
- `addClueRecord()` / `addAnnotation()` 写入全局记录池，供后续章节继续显示。
- 对话消息写入 `engine.notebookChatsByChapter[chapterKey]`，每章独立；切换章节后只显示本章对话。

## 完成标准

- 第一章进入时背景先出现，随后才出现对话框文字。
- 第一章笔记本“记录”页能看到序章已写入记录。
- 后续新增记录会继续保留到下一章。
- 对话内容按章节隔离。
- `npm run build` 通过。
