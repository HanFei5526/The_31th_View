# 改动计划：修复立绘切换/消失时的左移与瞬移问题

此计划旨在解决对话立绘（如沈念立绘）在切走或消失时向左平移/瞬移淡出的视觉问题，使立绘能够原地淡化消失。

## 问题原因分析

1. **坐标切换与不透明度过渡不同步**：
   在沈念的特定对话中，立绘带有 `portrait-reach` 类，该类将容器的定位由 `left: 0` 移动至 `left: 4%`。
2. **瞬移与淡出并发**：
   当对话切入旁白或结束时，JS 同时移除了 `visible`（控制淡出）和 `portrait-reach`（控制左边距恢复为 0）。
   由于 CSS 中仅对 `opacity` 定义了过度动画（`transition: opacity 0.3s ease`），导致 `left` 在淡出过程中立刻从 `4%` 跳变回 `0`。这种瞬间的左跳在 0.3s 淡出过程中被眼睛捕捉，表现为立绘“向左平移/瞬移后才消失”。

## 实施方案

我们通过纯 CSS 延迟（transition-delay）机制来解决此问题：
* 在立绘显示（添加 `.visible`）时，让 `left` 的更改立刻生效（0s 延迟）。
* 在立绘消失（移除 `.visible`）时，让 `left` 的更改延迟 0.3s 生效。这能保证在不透明度淡出的 0.3s 期间，立绘位置牢牢保持在原有位置，在完全隐形后才复位，从而实现**纯粹的原地淡化消失**。

### [MODIFY] [narration-bar.css](file:///e:/%E4%B8%AA%E4%BA%BA%E8%B5%84%E6%96%99/wzx/%E6%A1%8C%E9%9D%A2/%E5%8D%85%E4%B8%80%E6%99%AF/Web/src/styles/narration-bar.css)

修改立绘容器的过渡行为：

```diff
 .narration-portrait-container {
   position: fixed;
   bottom: 0;
   left: 0;
   pointer-events: none;
   opacity: 0;
-  transition: opacity 0.3s ease;
+  transition: opacity 0.3s ease, left 0s 0.3s;
   z-index: 99;
 }
 
 .narration-portrait-container.portrait-reach {
   left: 4%;
 }
 
 .narration-portrait-container.visible {
   opacity: 1;
+  transition: opacity 0.3s ease, left 0s 0s;
 }
```

## 影响范围

* 仅修改 `Web/src/styles/narration-bar.css` 中的立绘过渡样式。
* **不修改任何无关的 JS 或 CSS 代码**。
* 此修改可完美保持立绘消失时的原地淡出体验。

## 验证计划

1. 修改后在本地运行开发服务器。
2. 触发第一章沈念的对话并点击通过。
3. 仔细观察立绘在切为旁白（无立绘）时，是否为原地渐隐淡出，不再出现任何向左瞬移或滑动的现象。
