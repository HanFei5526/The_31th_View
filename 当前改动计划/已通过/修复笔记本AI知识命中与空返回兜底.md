# 修复笔记本 AI 知识命中与空返回兜底

## 背景

AI 后端代理与 API Key 配置已经验证通过：后端健康检查可用，页面内“修复笔记本”也能正常获得 AI 回复。

但在 Chrome 实测中发现一个新的体验问题：

- 玩家询问“装裱接缝残角指向什么”时，曾出现半句回复：“（翻到一条批注）装”
- 再次询问时，曾出现“（模型返回空文本……）”
- 询问基础资料“文徵明是谁”时可以稳定返回完整回答

这说明问题不在 API Key 或代理链路，而在“笔记本记录文本、知识片段、系统提示、空返回处理”之间的衔接。

## 问题判断

当前笔记本 prompt 中有严格约束：

1. 只能基于【当前可用知识】回答。
2. 【当前进度摘要】只用于判断资料是否解锁，不能作为可扩写资料来源。

但玩家在 UI 里实际看到并提问的内容，很多存在于 `engine.notebookRecords`，例如：

- `[线索] 装裱接缝残角 — 旧题签被刻意裁去，只留被覆盖的一角`

目前 `getAIContext()` 没有把 `notebookRecords` 作为“当前可引用记录”传入 prompt。于是玩家用“记录页标题”提问时，模型需要在“题签功能说明”等知识片段里间接匹配，稳定性不足。

## 修改目标

1. 让 AI 明确知道“已经写入修复笔记本记录页的内容”属于当前可引用依据。
2. 保持知识边界：只能引用已获得记录和已解锁知识，不能提前透露后续章节或终章真相。
3. 避免直接把“模型返回空文本……”暴露给玩家。
4. 对短截断、空返回做一次轻量重试或自然兜底。
5. 保持改动小范围，不重构 AI 架构。

## 涉及文件

预计涉及：

- `Web/src/core/game-engine.js`
  - 在 AI 上下文中加入 `notebookRecords` 快照。

- `Web/src/core/ai-prompts.js`
  - 给笔记本 prompt 增加“当前笔记本记录页”区块。
  - 调整约束：AI 可以基于【当前可用知识】和【当前笔记本记录页】回答。
  - 明确记录页只代表玩家已获得线索，不允许据此外推未解锁结论。

- `Web/src/core/ai-service.js`
  - 给 `queryNotebook()` 增加更低温、更稳定的参数。
  - 对空文本、过短异常文本做一次重试。
  - 重试后仍失败时，返回笔记本风格兜底文案，而不是“模型返回空文本……”。

可能涉及：

- `Web/src/data/knowledge-snippets.js`
  - 仅在必要时补少量同义标题，例如“装裱接缝残角”对应“旧题签残角/题签功能说明”。
  - 不新增未解锁真相。

## 主要改法

### 1. AI 上下文补充记录页

在 `getAIContext()` 中加入：

- `notebookRecords: [...this.notebookRecords]`

只传文本和类型，不传聊天历史，避免把玩家随口猜测当成事实依据。

### 2. Prompt 增加“当前笔记本记录页”

在 `buildNotebookQueryPrompt()` 中格式化记录页：

```text
【当前笔记本记录页 · 玩家已经获得的记录】
- [线索] 装裱接缝残角 — ...
- [线索] “……所见”残字 — ...
```

并更新约束：

- 可以基于【当前可用知识】和【当前笔记本记录页】回答。
- 记录页只说明“玩家已经获得了这些线索/物件/结论”。
- 不得用记录页推导未解锁章节真相。
- 如果两者都没有依据，仍回复“（翻了翻，没有找到相关记录）”。

### 3. 空返回与短异常兜底

在 `_callAPI()` 或 `queryNotebook()` 层处理：

- `content` 为空：记录错误，允许一次重试。
- 内容极短且明显不是合法边界回复，例如只有 1-3 个汉字：允许一次重试。
- 重试仍失败：返回 `（翻了翻，相关记录没有形成清晰答复。可以换个线索名再问一次。）`

避免玩家看到技术性文案：

- 不再在普通笔记本问答里显示“模型返回空文本……”

### 4. 参数更稳

笔记本查阅建议：

- `temperature: 0.2`
- `max_tokens: 220-300`

减少半句、跑偏和不稳定表达。

## 验收测试

后端直连：

1. `/api/health` 返回 `configured: true`
2. `/api/chat` 最小连通性返回正常

Chrome 页面实测：

1. 进入第二章，打开修复笔记本。
2. 提问：`请根据当前记录，用一句话说明“装裱接缝残角”指向什么。`
   - 预期：能回答旧题签残角/来源信息被遮蔽，不出现半句或空文本。
3. 提问：`“所见”残字说明什么？`
   - 预期：能回答观看记录/旁注相关，不剧透王蘅完整身份。
4. 提问：`底层细线可能是什么？`
   - 预期：只能说可能是定位痕迹，不能提前说完整低位视角真相。
5. 提问：`终章真相是什么？`
   - 预期：拒绝提前透露，返回当前资料不足或没有相关记录。

构建验证：

- `npm run build`

## 不做内容

- 不修改 API Key 配置方式。
- 不扩大菜单解锁逻辑。
- 不新增后续章节真相到早期知识片段。
- 不改变研讨门槛的关键词判定逻辑。
- 不重构整个 AI 服务，只做笔记本问答稳定性修复。

## 完成标准

- 已获得记录可以稳定被 AI 识别和引用。
- 未获得信息仍被知识边界挡住。
- 空返回、短截断不再直接暴露给玩家。
- Chrome 实测通过。
