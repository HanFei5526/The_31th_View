# 改动计划：第一章芙蓉榭光点、断簪与水面分界线优化

## 修改意图

芙蓉榭倒影谜题目前有三处视觉问题：

1. 栏杆交互光点偏到屋檐/建筑上方，需要落到真实栏杆位置。
2. 断簪图标细节偏复杂，缩小后不够清楚，需要改成和物件匣笔记本一致的简洁线条示意。
3. 水面分界线现在是一条横向 CSS 光带，和背景图真实岸线、平台边缘、石块遮挡关系融合不足，需要改成沿画面真实水岸分段出现的柔和光线。

## 修改方案

### 1. 栏杆光点位置

调整 `Web/src/pages/chapter1-paint.js` 中芙蓉榭场景的栏杆热点坐标：

| 热点 | 当前坐标 | 建议坐标 | 说明 |
| :--- | :--- | :--- | :--- |
| 栏杆 `_realRailing` | `(50, 35)` | 约 `(44.5, 56.5)` | 移到画面中部偏左、临水平台上方的栏杆处 |

实施时会根据浏览器实际预览微调到栏杆横梁附近，保持点击热区可用。

### 2. 断簪图标简化

同时简化两处断簪图形：

- 场景内倒影/翻转后的可点击断簪：改成一根斜向簪身 + 一个简单断口 + 一点微弱金光。
- 物件匣里的断簪图标：改成和修复笔记本图标一致的单色线条 SVG，不再画复杂花头和散落断尾。

暂不建议为断簪使用图片素材。当前物件匣已经是线条 SVG 体系，断簪、草图拓片、信件这类小物件如果统一用简洁 SVG，会更清楚、更轻，也方便后续复用和统一色彩。只有当后续需要在剧情里展示“物件大图/特写鉴赏”时，再单独做正式透明 PNG/WebP 素材。

### 3. 水面分界线分段路径

将 `.furong-waterline` 从整条横向渐变光带改成 SVG 路径层：

- 保留一个 48px 以上的透明点击区域，保证原来的点击和键盘交互不受影响。
- 在内部加入 `viewBox="0 0 100 100"` 的 SVG，用百分比坐标绘制多段路径。
- 路径大致沿以下区域分段：
  - 左侧太湖石与水面交界；
  - 中左平台/栏杆外侧的水岸短线；
  - 中部平台下沿与水面交界；
  - 右侧太湖石边缘水线。
- 每段使用淡金色低透明描边、轻微 blur 和渐隐动画，避免抢过原图墨线。

这种做法不需要替换背景图，也不需要引用临时截图。路径会随背景图比例缩放，适合当前 2386×1342 的正式芙蓉榭背景。

## 影响范围

- `Web/src/pages/chapter1-paint.js`
  - 调整芙蓉榭栏杆热点坐标。
  - 替换水面线 DOM 结构为可点击 SVG 分段路径。
  - 简化场景内断簪 SVG。
- `Web/src/styles/index.css`
  - 更新 `.furong-waterline` 及其内部 SVG/路径样式。
  - 更新 `.furong-hairpin` 图标样式。
- `Web/src/components/inventory-popup.js`
  - 简化物件匣内断簪图标。

不改剧情台词，不改谜题状态机，不改物件获得逻辑，不触及三条剧情基准。

## 验证计划

1. 启动前端并进入第一章芙蓉榭场景。
2. 确认栏杆光点落在栏杆横梁附近，点击后仍触发栏杆反馈。
3. 确认倒影中的断簪和翻转后的断簪都更简洁、可辨认，点击流程正常。
4. 点击水面分界线前后，确认提示、翻转动画、拾取断簪流程均正常。
5. 在桌面宽屏和较窄窗口下检查水面分段线是否仍贴近背景图，不出现突兀横条。
