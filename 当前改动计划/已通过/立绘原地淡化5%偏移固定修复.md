# 改动计划：修复立绘切入时的滑动问题并将偏移量调整至 5% 且完全固定

根据用户的反馈，我们决定保留立绘的偏移逻辑，但彻底消除淡入淡出时的所有平移滑动效果，让立绘在偏移后的新位置上“完全固定、原地淡入淡出”；同时将偏移量从 `4%` 调整为 `5%`。

## 问题原因分析

目前虽然通过 CSS 延迟在淡出时避免了左移，但在图片加载（淡入）过程中，浏览器对过渡状态切换的误判依然触发了 `left` 属性的滑入补间。

## 实施方案

为了达到“在偏移位置完全静止、原地淡入淡出”的效果，我们从根本上取消 CSS 中对 `left` 属性的任何过渡动画支持，并改用纯 JS 状态延迟方案：

1. **取消 CSS 过渡中对 `left` 的任何约束**：
   在 `narration-bar.css` 中只保留 `transition: opacity 0.3s ease`，去掉一切与 `left` 相关的过渡。任何时刻 `left` 的改变都是瞬间（0秒）发生。
2. **偏移量由 4% 调整为 5%**：
   修改 `.narration-portrait-container.portrait-reach` 的 `left` 为 `5%`。
3. **JS 端延迟移除 `portrait-reach` 样式类**：
   在 `narration-bar.js` 中的 `setPortrait(null)` 隐藏立绘分支里，只移除 `.visible` 触发原地淡出，**不立刻移出** `.portrait-reach`（使其在淡出的 0.3s 期间及之后保持在 5% 位置不动，保持原地渐隐）。下一次传入新立绘时，在不透明度为 0 的状态下由 JS 根据新立绘重新判定并瞬间置回 0 偏置，从而让用户在视觉上完全感受不到任何滑动。

## 影响范围

* 仅影响 `narration-bar.css` 与 `narration-bar.js` 的立绘相关渲染定义，无其他逻辑影响。
