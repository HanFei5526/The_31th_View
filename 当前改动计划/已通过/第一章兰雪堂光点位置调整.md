# 改动计划：调整第一章兰雪堂场景的点击光点位置

此计划旨在根据用户提供的标注图，精细调整第一章画中世界（兰雪堂场景）中四个交互光点（Hotspots）的坐标位置，以确保其显示及交互逻辑符合美术视觉设计。

## 修改意图与方案

原有坐标在适配背景图时存在一定偏差（如匾额光点偏高至屋顶、廊柱光点偏右至太湖石上）。为使玩家在点击时感到视觉精准，我们需要根据用户标注的红圈位置，将各个光点调整至对应的核心景物上。

我们在原图 `2666 x 1499` 像素比例上绘制并比对了坐标点，确定了以下最佳的百分比位置：

| 光点名称 (Hotspot ID) | 原坐标 (x%, y%) | 新坐标 (x%, y%) | 景物对应位置 |
| :--- | :--- | :--- | :--- |
| **匾额 (plaque)** | `(50, 22)` | `(50, 38)` | 从瓦片区域下移至黑框白底的“蘭雪堂”匾额“雪”字下沿 |
| **翠竹 (bamboo)** | `(15, 50)` | `(16, 60)` | 从左侧芭蕉叶上边缘下移至芭蕉树的主干顶端分叉处 |
| **廊柱 (pillar)** | `(70, 55)` | `(61, 63)` | 从右侧太湖石顶部下移至左侧门扇与廊柱交界处 |
| **青石板路 (stonePath)** | `(50, 85)` | `(42, 93)` | 从底部的砚台纸张旁下移至青石板路的最下一排偏左位置 |

## 影响范围

* 仅修改 `Web/src/pages/chapter1-paint.js` 文件中 `_buildLanxueScene` 方法内的四个 `_createHotspot` 坐标值。
* **不修改任何无关代码**。
* **无其他系统或逻辑影响**。

## 验证计划

1. 修改后，在浏览器中启动开发服务器（`npm run dev`）。
2. 进入第一章兰雪堂场景，验证四个光点的位置是否完美贴合用户要求的红圈位置。
3. 验证点击每个光点时是否能正常触发对应的旁白或线索解锁逻辑。
