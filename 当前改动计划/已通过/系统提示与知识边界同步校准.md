# 系统提示与知识边界同步校准

## 状态

已完成。

## 检查目标

- 检查系统提示、线索记录、物件获得、AI 知识解锁之间的顺序是否一致。
- 确认是否存在“还没探索，就已经确认获得线索或物件”的问题。
- 明确后续统一规则：提示只负责引导，记录和进度标记才负责解锁 AI 知识。
- 校准笔记本自动展开与自动收起时机，避免探索画面被持续遮挡。

## 当前结论

整体流程基本符合预期：大多数关键线索和物件都是在玩家已经点击、看见、读到或完成操作后，才写入记录并更新进度。

需要处理一个小边界点：

- 第三章 `sketchRevealed` 当前在墙面草图图片出现后立刻写入进度，但关于“这不是正式画作，而是草图”的叙事解释在随后播放。严格来说，知识片段 `ch3_sketch_revealed` 应等解释性叙事完成后再解锁。

需要在文档中明确一个设计例外：

- `ensureCarryoverForChapter()` 会在进入后续章节时补齐序章基础线索、综合结论和笔记本。这是章节入口兜底/旧存档兼容，不属于普通探索中的“提前获得线索”。

笔记本自动展开/收起检查结论：

- 第一章断簪轻量讨论、第三章信件后讨论、序章综合研讨、终章记录回顾的自动展开时机合理。
- 第二章轻量讨论结束后未主动收起笔记本，进入小飞虹探索时仍占画面，建议收起。
- 第二章普通场景入口存在自动展开笔记本，建议改为只更新快捷问题和系统提示，保留玩家手动打开入口。

## 建议统一规则

### 1. 系统提示不直接解锁知识

普通系统提示、渐进提示、探索提示只告诉玩家下一步可以做什么，不应改变 AI 知识边界。

例如：

- “这里似乎还有东西可以探索。”
- “也许物件匣中有能帮上忙的东西。”
- “试试后退一步，蹲下来看。”

这些提示不应直接解锁新知识。

### 2. 线索确认才解锁知识

只有玩家实际看见线索，并写入笔记本记录或进度标记后，AI 才能解释对应内容。

标准顺序：

```text
玩家点击 / 操作
→ 叙事文本揭示线索
→ 写入 gameProgress 标记
→ 写入笔记本记录
→ 系统提示“已记录线索”
→ 展开笔记本 / 显示快捷问题
→ AI 可基于新边界回答
```

### 3. 物件获得才解锁物件知识

只有玩家已经拾取、查看或阅读物件后，才设置 `hasXxx = true`，并允许 AI 讨论该物件。

标准顺序：

```text
玩家发现物件
→ 叙事展示外观 / 文字 / 关键细节
→ collectItem()
→ 设置 hasXxx = true
→ addClueRecord()
→ 系统提示“获得物件”
→ 显示相关快捷问题
```

### 4. 章节入口兜底单独处理

章节入口补齐进度只用于：

- 旧存档兼容。
- 跳章节调试。
- 保证后续章节不会缺失前置基础资料。

它不代表普通流程中玩家可以跳过探索直接获得线索。

### 5. 笔记本展开/收起规则

- 普通探索：画面优先，不自动展开笔记本；仅更新快捷问题和系统提示。
- 轻量讨论：自动展开笔记本，讨论完成或跳过后自动收起。
- 综合研讨：自动展开并锁定笔记本，通过后解锁，进入下一阶段前收起。
- 终章回顾：自动展开到记录页，点击“开始复原”后收起。
- 玩家手动打开：普通探索中尽量尊重玩家操作，不主动关闭；切入转场、强叙事或系统讨论结束时可以收起。
