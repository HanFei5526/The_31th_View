# 《卅一景》

**文人悲情 / 双世界解谜 / 园林视角探秘**

一款以苏州拙政园与文徵明《拙政园三十一景图》为叙事支点的 Web 互动叙事游戏。

你将扮演古画修复研究生**沈念**，在导师周鹤年的指导下，对一幅传世册页进行数字化修复。但你很快发现，画中藏着不该存在的视角——一个从未被记录的观看者的痕迹。

当你"跌入"画中拙政园，现实与画境开始交替。你要找回的不是一幅失踪的画，而是一段被正式作品体系吸收、遮蔽了五百年的观看来源。

> 王蘅为虚构人物。本作品基于真实文化背景进行虚构创作，不作为历史考证文本。

## 游戏流程

```
序章 · 残页 → 第一章 · 东园 → 第二章 · 中园 → 第三章 · 西园 → 终章 · 第三十一景
```

每章在**画中世界**（拙政园）与**现实工作室**之间切换。终章设有三个结局。

## 游戏特色

- **双世界交替**：现实修复工作室与画中拙政园，两种视觉风格、两套交互逻辑
- **古画修复交互**：使用放大镜、纸质分析、侧光照射等工具检测古画，寻找隐藏线索
- **自由表达式研讨**：不做选择题，用自己的话表达对线索的理解，AI 判定推理是否合理
- **轻量 RAG 知识系统**：AI 助手按游戏进度逐步解锁知识，不会提前剧透
- **诗词比对谜题**：在竖排古诗中逐字寻找异文差异
- **水墨风转场**：墨迹扩散、卷轴展开、渗字效果等沉浸式动画

## 快速开始

### 环境要求

- Node.js 18+
- npm

### 启动前端

```bash
cd Web
npm install
npm run dev
```

浏览器访问 `http://localhost:5173` 即可开始游戏。

### 启用 AI 功能（可选）

游戏内置离线降级模式，不配置 AI 也可完整体验所有章节。如需 AI 对话功能：

1. 复制环境变量模板：

```bash
cd Web/server
cp .env.example .env
```

2. 在 `.env` 中填入你的 DeepSeek API Key：

```
DEEPSEEK_API_KEY=你的Key
```

3. 启动 AI 代理服务：

```bash
npm install
npm start
```

AI 代理运行在 `localhost:8787`，前端会自动通过 `/api` 代理连接。

### 构建

```bash
cd Web
npm run build     # 生产构建，输出到 dist/
npm run preview   # 本地预览构建产物
```

## 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Vite + 原生 JavaScript / CSS，无框架依赖 |
| 后端 | Node.js + Express（仅作 AI API 代理） |
| AI | DeepSeek API |
| 存储 | 浏览器 localStorage |

## 项目结构

```
Web/
├── index.html              # 入口页面
├── vite.config.js          # Vite 配置
├── server/                 # AI 代理后端
├── public/images/          # 游戏图片资源（按章节组织）
└── src/
    ├── main.js             # 应用入口
    ├── core/               # 游戏引擎、场景管理、AI 服务、存档等核心系统
    ├── data/               # 知识片段数据
    ├── pages/              # 各章节场景
    ├── components/         # UI 组件（古画查看器、对话框、笔记本等）
    └── styles/             # 样式文件
```
