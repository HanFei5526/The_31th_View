# UI重构：叙事层与AI对话层分离

## Context

当前 PrologueDock 用一个容器混装了三种内容：旁白/脚本台词（被动观看）、AI研讨对话（主动输入）、系统反馈。导致：
- 玩家分不清"在看故事"还是"该动脑子"
- 脚本播放时输入框禁用，与"AI任何时候可用"矛盾
- 旁白和AI回复混在同一时间线，回滚时无法区分

**决策：**
- AI 任何时候都可交互（包括脚本播放期间）
- 叙事和对话物理分离为两个独立通道
- 底部对话框保留当前古色古香风格
- 游戏画面 100% 全屏，AI 笔记本面板为悬浮面板（可收缩/展开）
- 右下角 HUD 横排按钮：📓 笔记本 + 📦 物件匣（💾 存档 + ⚙️ 设置后续章节再加）
- 笔记本面板内 tab 切换（对话 / 记录）
- 工具按钮在笔记本面板底部（扫描阶段可见，用完收起）

## 三大通道定义

### 通道一：叙事对话框（NarrationBar）

| 属性 | 说明 |
|------|------|
| 位置 | 画面底部全宽（100%，HUD按钮在对话框右侧同行） |
| 风格 | **保留当前古色古香风格**（暗木匾额底色、金色文字） |
| 内容 | 旁白文本、人物脚本台词（预写） |
| 交互 | 点击可跳过打字机 / 加速推进，**无输入框** |
| 生命周期 | 显示→停留→点击推进，不保留历史 |
| 立绘 | 人物说话时，半身立绘出现在对话框上方（经典VN布局） |
| 角色区分 | 人物台词=左上角显示人物名字+立绘，旁白=无名字无立绘+弱化样式 |

**人物台词态：**
- 左上角：人物名字（如"周鹤年"），暖金色
- 文字：左对齐，正常亮度
- 立绘：半身像出现在对话框上方左侧，底部紧贴对话框顶部

**旁白态（同框不同样式）：**
- 无人物名字、无立绘
- 对话框底色更暗
- 文字居中，字色偏淡（如 `#a89878`）
- 边框虚化或去除

### 通道二：悬浮笔记本面板（NotebookPanel）

**面板身份：沈念的修复笔记本**

| 属性 | 说明 |
|------|------|
| 身份 | 非人格化的修复笔记本（不是一个人在说话） |
| 内容来源 | 参考文献、修复记录、学术资料 + **周鹤年之前写的批注** |
| 口吻示例 | "（参考文献摘录）""（翻到某页）""（周老师的批注）" |
| 全场景通用 | 现实世界和画中世界都合理 |
| 引导方式 | 通过"周老师批注"的间接引语做推理引导 |

**与对话框的区分：**
- 对话框的周鹤年 = 当面在说话（实时、直接引语）
- 笔记本中的周鹤年批注 = 他之前写在本子上的（历史记录、间接引语）

**面板行为：**
| 状态 | 说明 |
|------|------|
| 脚本阶段（hasNotebook=false） | 面板隐藏，HUD 隐藏（纯全屏叙事） |
| 获得笔记本后（hasNotebook=true） | HUD 出现（📓📦），笔记本可点击展开 |
| 探索阶段（扫描+寻线） | 面板强制展开不可收缩，HUD 中📓灰色禁用 |
| 综合门槛时 | 面板隐藏，HUD 隐藏，GatePanel 全屏接管 |
| 后续章节 | 面板可收缩/展开，HUD 正常显示 |

**面板内部结构：**
- 顶部：tab 切换（"对话" / "记录"）
- 对话 tab：AI 对话历史 + 快捷按钮 + 输入框
- 记录 tab：已发现线索列表 + 沈念批注时间线
- 底部（仅扫描阶段）：工具区（三个按钮 + 使用计数）

### 通道三：综合门槛（GatePanel）

| 属性 | 说明 |
|------|------|
| 触发 | 三线索集齐后 |
| 形态 | 全屏面板，无关闭按钮，必须通过 |
| 身份 | 周鹤年面对面讨论（直接对话态，非笔记本） |
| 与笔记本面板关系 | 面板隐藏，GatePanel 接管全部AI交互 |
| 通过后 | GatePanel 卸载，笔记本面板恢复 |

### HUD 按钮栏

| 位置 | 右下角横排 |
|------|---------|
| 当前按钮 | 📓 笔记本（展开/收缩面板）+ 📦 物件匣 |
| 后续扩展 | 💾 存档 + ⚙️ 设置（多章节串联后再加） |
| 样式 | 半透明小圆形图标，hover 时提示文字 |
| 物件匣 | 点击弹出浮层查看已收集物件及描述 |

## AI面板引导设计

### 三层引导机制

**第一层：输入框 placeholder（始终存在，按阶段动态变化）**

| 阶段 | placeholder |
|------|------------|
| 脚本/对话阶段 | "翻阅笔记本……" |
| 工具扫描阶段 | "这个工具能看出什么？" |
| 自由探索（未找到线索） | "这幅画有什么不对劲的地方？" |
| 发现 1-2 条线索后 | "这条线索意味着什么？" |
| 三线索集齐后 | "三处痕迹之间有什么联系？" |

**第二层：快捷按钮（关键节点刷新）**

| 触发时机 | 快捷按钮示例 |
|----------|-------------|
| 面板首次展开（脚本阶段） | "拙政园三十一景是什么？""修复笔记本里有什么？" |
| 进入扫描阶段 | "放大镜能看到什么？""纸质分析有什么用？" |
| 发现线索 A 后 | "装裱接缝说明什么？""旧题签为什么被裁？" |
| 发现线索 B 后 | "'所见'是什么意思？""边注通常记录什么？" |
| 发现线索 C 后 | "这条线为什么这么低？""辅助线是做什么用的？" |
| 三线索集齐（进门槛前） | "三条线索有什么共同点？""谁在遮盖这些痕迹？" |

**按钮行为：**
- 点击后该按钮变灰（标记已问），其余保留
- 阶段切换时整组刷新为新按钮

**第三层：空状态引导语（仅面板无对话历史时显示）**

> （笔记本扉页）
> 这是你的修复笔记本。随时翻阅查找参考信息，或记录你的发现。
> 周老师的批注散落在各页之间。

对话开始后引导语消失，不再出现。

### AI回复末尾追问

笔记本回复后可附一句周老师批注作为追问引导，推动玩家继续深入思考。

**格式：** 回复正文 + 换行 + `（周老师的批注：反问句）`

**示例：**
- 玩家问"装裱接缝说明什么？"
- 笔记本回复：「（参考文献摘录）二次揭裱常见于修复重装的册页中，其目的通常是替换受损的装裱材料或调整尺幅。」
- 追问引导：「（周老师的批注：如果画心本身没问题，为什么还要动装裱？除非装裱下面藏着什么。）」

**约束：**
- 追问不给出答案，只用反问引导方向
- 不透露后续章节信息
- 不突破当前章节的认知范围

## 改动文件清单

### 1. 新建 `Web/src/components/narration-bar.js`

替代 PrologueDock 的脚本播放功能。纯叙事对话框，无输入能力，支持立绘。

**接口设计：**
```
class NarrationBar {
  mount(root)
  playLine(speaker, text, options?): Promise<void>
    — speaker: 人物名(如"周鹤年") 或 null(旁白)
    — options.portrait: 立绘图片路径（可选）
    — 打字机播放，完成后等待点击推进
  showFeedback(text): Promise<void>       — 短反馈（工具使用后的提示），自动消失
  setPortrait(src | null)                  — 切换/隐藏立绘（不换台词时单独换表情用）
  dismiss()                                — 立即关闭当前对话
  unmount()
}
```

**视觉规格（保留古色风格）：**
- 位置：画面底部全宽
- 沿用当前 PrologueDock 的暗木匾额底色
- 人物台词：左上角显示名字（暖金色 `#d4a853`），文字左对齐
- 旁白：无名字，底色更暗，文字居中，字色偏淡 `#a89878`
- 立绘：半身像，对话框上方左侧，底部紧贴对话框顶部，淡入/淡出 0.3s
- 打字机速度：42ms/字（沿用当前）
- 点击可跳过打字机或推进到下一句

### 2. 新建 `Web/src/components/notebook-floating.js`

悬浮笔记本面板。替代旧 PrologueDock 的研讨模式和旧 notebook-panel。

**接口设计：**

class NotebookFloating {

  // === 对话 tab ===
  showNPCMessage(text)
  showPlayerMessage(text)
  showSystemMessage(text)
  setLoading(bool)
  clearHistory()
  showQuickThoughts(thoughts[])
  hideQuickThoughts()
  setPlaceholder(text)

**视觉规格：**

## 实施注意事项

### z-index 层级约定

| 元素 | z-index | 说明 |
|------|---------|------|
| 游戏画面（背景/古画） | 基层 (auto) | 不设 z-index |
| NarrationBar（对话框+立绘） | 100 | 始终在画面之上 |
| HudBar（📓📦按钮） | 150 | 在对话框之上 |
| NotebookFloating（笔记本面板） | 200 | 悬浮在所有常规 UI 之上 |
| 线索弹窗遮罩 | 500 | 冻结画面操作 |
| GatePanel（全屏门槛） | 999 | 最高层，接管一切 |

### prologue.js 状态机保持

当前 prologue 的阶段状态机不变，只替换每个状态对应的 UI 调用方式：

```
NARRATION → DIALOGUE → SCANNING → EXPLORATION → CLUE_POPUP → SYNTHESIS_GATE → CONVERGENCE → TRANSITION → COMPLETE
```

- NARRATION / DIALOGUE：改用 NarrationBar.playLine()
- SCANNING：改用 NotebookFloating.showToolSection()
- EXPLORATION：NotebookFloating 强制展开
- SYNTHESIS_GATE：NotebookFloating.hide() + GatePanel 全屏
- CONVERGENCE / TRANSITION / COMPLETE：复用现有逻辑
