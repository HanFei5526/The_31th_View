## 一、双世界界面风格

| 世界 | 风格 | 强调元素 |
|---|---|---|
| **现实世界**（修复工作室） | 现代简洁，冷色调 | 扫描仪、数据库、报告草稿、档案倒计时 |
| **画中世界**（明代拙政园） | 水墨淡彩，暖旧色调 | 纸纹、墨迹、倒影、题诗、残字、低位视角 |

## 二、世界切换与转场

- 序章结束后，玩家“跌入”画中世界；此后每章结束时自动返回现实世界。
- 现实世界中提供“继续修复”按钮进入下一章。

**跌入转场（序章→画中）动效**：

> 冷蓝色的扫描界面渐渐褪去。纸纹变成地面，墨线变成栏杆，边缘那一枚残字像水中落叶一样沉了下去。

- 触发后：屏幕画面开始流动，墨迹如水波扩散；玩家视角被吸入表层画面与边缘残字之间的缝隙。

**章末返回（画中→现实）动效**：画面褪色，玩家“醒回”工作室。

## 三、各谜题交互 / UI 呈现层

### 序章 · 工具扫描 + 自由探索

- 古画全屏展示，右侧悬浮笔记本面板（360px），底部叙事对话框（55%宽居中）。
- 三个工具按钮位于笔记本面板底部工具区：🔍 放大镜 / 🔬 纸质分析 / 💡 侧光照射。
- 工具仅 filter 滤镜效果，无画面位移；首次使用显示 pv-feedback 检测结果（4.5s 消失），按钮标记 ✓。
- 已使用工具不可重复点击；三工具全部使用后自动切放大镜，工具区锁定灰掉。
- 自由探索阶段支持滚轮缩放 / 拖拽平移 / 点击寻找线索。
- 发现线索即时确认并记入笔记本，同时启动辅助讨论（周老师批注浮现）。
- 渐进提示：每 3 次错误多给一条呼吸光斑，文案按具体线索方位差异化。
- 三处线索集齐后，玩家点击画面下方"进入综合研讨"按钮触发综合门槛。
- 综合研讨在笔记本面板内进行（Tab 改名"综合研讨"），通过后结论记入笔记本。
- 研讨通过后触发汇聚动画 → 点击交会点 → 跌入画中转场。
- 详细交互规格见 `分章节实现文档/序章_残页_玩法设计.md`。

### 第一章 · 水面倒影

- 屏幕上半部分是真实芙蓉榭，下半部分是水面倒影。
- 真实栏杆空无一物，倒影栏杆挂着断簪；玩家需点击倒影中的物件。
- 关键控件：**“镜像视角 / 翻转”按钮**——点击后画面翻转，倒影变正像，断簪可拾取。
- 屏幕边缘“翻转”图标在提示升级时闪烁。

### 第二章 · 诗词比对

- 屏幕分左右两栏：左栏画中题诗，右栏参考流传版本。
- 玩家逐字点击差异；差异字可高亮（但组合含义由玩家理解）。

### 第三章 · 草图显现

- 打开**物件匣**选用道具。
- 操作1：将残砚靠近墙面、用朱砂轻拂 → 灰泥表面浮现淡红色线条。
- 操作2：用断簪沿红线轻划 → 灰泥沿旧刻痕剥落，草图显现。
- 草图右下角凹槽点击 → 打开暗格取出油纸包。

### 终章 · 四问复原 + 结局选择

- 四问以选择题/点击地图/多选景物形式呈现，每答对一题画面浮现一部分景色。
- 结局阶段：三条路以三个选择按钮呈现，每个选项附“选择前犹豫提示”文案。

## 四、线索背包（物件匣）UI

- 物件匣常驻入口，存放：修复笔记本、断簪、残砚、王蘅的信、草图拓片。
- 每个物件可查看描述文本（flavor）与已记录线索。
- 第三章草图谜题需从物件匣中选取道具作用于墙面。
- 精细 UI 为可后置项（见 MVP 优先级🟢）。

## 五、AI 交互面板（待重构）

> 以下为即将实施的新方案，详见 `改动计划/UI重构_叙事层与AI对话层分离.md`。
> 旧实现对应 `Web/src/components/chat-panel.js`（deprecated）和 `Web/src/components/prologue-dock.js`（待替换）。

### 设计原则

- **叙事与 AI 交互物理分离**：两个独立通道，互不阻塞
- **AI 任何时候都可交互**：包括脚本播放期间
- **AI 面板身份统一为"修复笔记本"**：非人格化，全场景通用

### 双通道布局

```
┌──────────────────────────── 100% ────────────────────────────┐
│                                                               │
│                                              ┌──────────┐    │
│                                              │📓 修复笔记本│    │
│  ┌─────┐      游戏画面（全屏，无遮挡）        │[对话|记录] │    │
│  │ 立绘 │                                    │[对话历史]  │    │
│  │     │                                    │[快捷按钮]  │    │
│  │     │                                    │[输入框]    │    │
│  ├─────┴──────────────────────────────┐     │[工具区]    │    │
│  │ 周鹤年                              │     └──────────┘    │
│  │ "先看装裱层。"                       │  ┌───┐             │
│  └────────────────────────────────────┘  │📓│             │
│                                          │📦│             │
│                                          └───┘             │
└──────────────────────────────────────────────────────────────┘
 ←────── 对话框 ~80% ────────────────→  ←HUD→
                                         笔记本面板为右侧悬浮 ~240px
```

### 通道一：叙事对话框（底部）

- **位置**：画面底部居中，宽度 55%
- **风格**：保留当前古色古香暗木匾额风格
- **内容**：旁白 / 人物脚本台词（预写），无输入框
- **立绘**：人物说话时半身立绘出现在对话框上方（经典VN布局）
- **旁白态**：无名字、无立绘、底色更暗、文字居中偏淡
- **交互**：点击跳过打字机 / 推进下一句
- **无避让**：笔记本面板展开时对话框不位移，55%宽度保证不重叠

### 通道二：修复笔记本面板（右侧悬浮）

- **位置**：右侧悬浮 ~240px，叠加在游戏画面上（游戏画面仍为 100%）
- **显示时机**：`hasNotebook=true` 后通过 HUD 📓按钮展开
- **阶段行为**：脚本阶段隐藏 → 获得后可收缩/展开 → 探索阶段强制展开 → 门槛时隐藏 → 后续章节自由收缩
- **身份**：沈念的修复笔记本（非人格化）
- **内容来源**：参考文献、修复记录 + **周鹤年之前写的批注**（间接引语）
- **口吻**：「（翻到某页）」「（参考文献摘录）」「（周老师的批注）」
- **上方**：AI 对话区（对话历史 + 快捷按钮 + 输入框）
- **下方**：工具区（仅扫描阶段可见，用完收起）
- **引导机制**：
  - 输入框 placeholder 按阶段动态变化
  - 快捷按钮在关键节点刷新（点击变灰，其余保留）
  - AI 回复末尾可附"（周老师的批注：反问句）"追问引导
  - 空状态时显示笔记本扉页引导语

### 综合门槛（全屏 GatePanel）

- **触发**：三线索集齐后
- **形态**：全屏面板，无关闭按钮，必须通过
- **身份**：周鹤年面对面讨论（直接对话态，非笔记本）
- **与右侧面板关系**：笔记本面板 `hide()` 隐藏，HUD 隐藏，GatePanel 接管全屏
- **通过后**：GatePanel 卸载，笔记本面板 `show()` + `expand()` 恢复

### 身份体系总结

| 通道 | 身份 | 周鹤年的存在方式 |
|------|------|----------------|
| 底部对话框 | 旁白 / 周鹤年脚本台词 | 当面说话（直接引语） |
| 右侧面板（日常） | 修复笔记本 | "周老师的批注"（间接引语） |
| GatePanel（门槛） | 周鹤年面对面讨论 | 直接对话（特殊仪式） |

## 六、交互通用规则

- **不设硬失败**：错误操作只触发反馈文字与提示升级，允许重新尝试。
- 关键音效：墨迹流动声、灰泥剥落声、扫描仪声（可简化）。
- 现实世界的“修复报告即将定稿”压力通过文本反复提示呈现，不做硬倒计时。
